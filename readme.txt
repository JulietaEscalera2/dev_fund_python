*REGLAS DE JUEGO*
- no se sube *nada* directo a develop
- cualquier pull request a develop se manda conmigo como code reviewer
- puedes crear cuantas branches quieras (todas tienen q salir de develop)
- lo mas recomendable, es q no trabajemos sobre un mismo archivo, pero, puedes tocar cualquier archivo, pero ante cualquier merge conflict, tu codigo sera borrado
- Best Practices dicen q no se deberia subir nada q no este corriendo, pero para nosotros eso puede variar por cosas de tiempo y demas, pero si o si cualquier cosa q pase a develop tiene q ser funcional y sin errores (mejor si ni warnings tiene)
